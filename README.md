# TCP-DCERL+: Delay and Congestion Estimation with Reinforcement-Like Logic

A custom TCP congestion control algorithm implemented as an ns-3 module.

---

## Overview

**TcpDcerl** (TCP Delay-Based Congestion Estimation with Reinforcement-Like Logic) is a research congestion control algorithm built on top of ns-3's `TcpCongestionOps` base class. It uses RTT (Round-Trip Time) measurements and two tunable flow parameters — internal load (`Bi`) and external load (`Be`) — to dynamically adjust the congestion window (cwnd) in response to network conditions.

The algorithm is inspired by delay-based congestion control principles, where RTT inflation relative to the minimum observed RTT serves as a signal for congestion.

---

## Algorithm Description

### Core Idea

DCERL+ distinguishes between two sources of congestion load:

- **`Bi` (Internal Load):** Represents load generated by the local flow itself.
- **`Be` (External Load):** Represents load from competing background traffic.

Using these two parameters alongside RTT measurements, the algorithm computes a ratio `R` that determines whether to grow or shrink the congestion window.

### Key Variables

| Symbol | Description |
|--------|-------------|
| `T` | Minimum observed RTT (baseline, approximates propagation delay) |
| `λ` (lambda) | Ratio of current RTT to minimum RTT (`rtt / T`) |
| `li` | Internal congestion signal: `(rtt − T) × Bi` |
| `le` | External congestion signal: `(rtt − T) × Be` |
| `lmax` | `max(li, le)` |
| `N` | Combined congestion estimate: `λ × lmax` |
| `R` | Decision ratio: `N / li` |
| `γ` (gamma) | Initial cwnd multiplicative factor (default: 2) |

### Window Update Rule

```
If li < N  (underutilized — grow):
    cwnd += max(1, R) × MSS

Else  (congested — shrink):
    ssthresh = max(cwnd/2, 2×MSS)
    cwnd -= max(1, 1/R) × MSS
```

The same rule is applied on detection of 3 duplicate ACKs (fast retransmit trigger).

### Event Handling

| Event | Action |
|-------|--------|
| Normal ACK | Compute DCERL+ rule and update cwnd |
| 3 Dup ACKs | Apply DCERL+ rule again, reset dup ACK counter |
| >20 Dup ACKs (timeout proxy) | Reset cwnd to `γ × MSS`, reset ssthresh to 65535 |

### RTT Tracking

RTT is tracked using an exponential moving average:

```
avgRtt = 0.9 × avgRtt + 0.1 × currentRtt
minRtt = min(minRtt, currentRtt)
```

## Integration with ns-3

### Placement

Copy both files into your ns-3 source tree:

```
src/internet/model/tcp-dcerl.h
src/internet/model/tcp-dcerl.cc
```

### Build System

Add the files to the module's `CMakeLists.txt` (or `wscript` for older ns-3 versions):

```cmake
# In src/internet/CMakeLists.txt
target_sources(internet PRIVATE
  model/tcp-dcerl.cc
)
```

### Usage in Simulation Script

```cpp
#include "ns3/tcp-dcerl.h"

// Set the congestion control algorithm globally
Config::SetDefault("ns3::TcpL4Protocol::SocketType",
                   TypeIdValue(TcpDcerl::GetTypeId()));

// Or set it on a specific socket
socketBase->SetAttribute("CongestionOps",
                         PointerValue(CreateObject<TcpDcerl>()));
```

## Parameters

The following parameters are set at construction time. To make them configurable via `ns3::TypeId` attributes, they can be registered in `GetTypeId()`.

| Parameter | Default | Description |
|-----------|---------|-------------|
| `m_Bi` | 1.0 | Internal load weight |
| `m_Be` | 1.0 | External load weight |
| `m_gamma` | 2 | cwnd multiplier used after timeout |
| RTT EMA α | 0.1 | Smoothing factor for average RTT |

## Design Notes and Limitations

- **Timeout detection is approximate.** The algorithm uses a duplicate ACK counter exceeding 20 as a proxy for a retransmit timeout. A more accurate implementation would hook into `TcpSocketBase`'s actual RTO callback.
- **Bi and Be are static.** In a full reinforcement-learning extension, these weights would be updated dynamically based on observed throughput or reward signals.
- **No slow start phase.** The algorithm operates solely via `IncreaseWindow`, meaning slow start is not explicitly modeled. Depending on ns-3 version, the base class may handle slow start separately.
- **RTT source.** The algorithm reads `tcb->m_lastRtt`, which reflects the most recently measured RTT sample. This may differ from smoothed RTT (SRTT) used in standard TCP implementations.
